Nixellar

Introduction: Please read about the 'Type of Service' you desire, to know more about it.
We facilitate 6 services. We have divided them into '3 Types' for user convenience.
Namely, these are:
  • from: XLM
    • to: Nixellar (Type A)
    • to: NIX (Type B)
  • from: Nixellar
    • to: XLM (Type C)
    • to: NIX (Type D)
  • from: NIX
    • to: Nixellar (Type E)
    • to: XLM (Type F)

Services available on Nixellar:

Type A : [XLM] into [Nixellar]

Users can buy 'Nixellar' on the SDEX using their native 'XLM', for example, from StellarPort.io, StellarX.com, StellarTerm.com etc. directly.
  • Click here for a comprehensive guide.

    Type B : [XLM] into [NIX]

    Firstly, users need to convert their "XLM" to "Nixellar" (asset-code=NIX & issuer= G...i). Then they may proceed to convert this asset into 'NIX'. Please undertake the following steps:
    1. It can be done by partaking "Nixellar Service : Type A" discussed above.
    2. Next, user needs to partake "Nixellar Service : Type D".
  • Click here for a comprehensive guide.

    Type C : [Nixellar] into [XLM]

    Users can buy 'XLM' on the SDEX using your 'Nixellar', for example, from StellarPort.io, StellarX.com, StellarTerm.com etc. directly.
  • Click here for a comprehensive guide.

    Type D : [Nixellar] into [NIX]

    Users need to return the 'Nixellar' they hold to the issuing account (address=G...i) on the Stellar blockchain, so that we may remove them from circulation. This is a relatively cheaper process.
    1. User needs to communicate their "NIX Platform address" to recieve 'NIX'.
    2. User has to send their 'Nixellar' to "Nixellar ('NIX') issuing acount, G...i.
    3. We send the 'NIX' to the users "NIX Platform address".
  • Click here for a comprehensive guide.

    Type E : [NIX] into [Nixellar]

    A user who already posses a Stellar blockchain account can choose "Route 1". We recommend "Route 2" and "Route 3" for users without a Stellar account. A user may choose either of the three proposed routes.
      Route 1:
    1. User sends NIX to our NIX-deposit-address, on the NIX Platform.
    2. User communicates their Stellar address for recieving Nixellar.
    3. User establishes trustline to Nixellar, on the Stellar blockchain.
    4. We send Nixellar to users Stellar address.
      Route 2:
    1. User generates a new Stellar Network keypair and communicates their address.
    2. User sends NIX to our NIX-deposit-address, on the NIX Platform.
    3. We send XLM on Stellar blockchain account for user.
        This includes:
      • Covering all minimum balance requirements.
      • Creating trustline to NIxellar.
      • Lock-in balance requirements for assets.
      • Little change for executing transactions on the Stellar Network.
    4. User establishes trustline to Nixellar, on the Stellar blockchain.
    5. We transfer Nixellar to the Stellar account created by user.
      Route 3:
    1. User sends NIX to our NIX-deposit-address, on the NIX Platform.
    2. We create a new Stellar blockchain account for user.
        This includes:
      • Covering all minimum balance requirements.
      • Creating trustline to Nixellar.
      • Lock-in balance requirements for assets.
      • Little change for executing transactions on the Stellar Network.
    3. User generates a new Stellar Network keypair and communicates their address.
    4. We transfer ownership of the account created by us.
  • Click here for a comprehensive guide.

    Type F : [NIX] into [XLM]

    Users need to convert their "NIX" to "NIX asset" - 'Nixellar' issued by G...i first. Then they may directly exchange their "Nixellar" on the Stellar DEX.
    1. It can be done by partaking "Nixellar Service : Type E" discussed above.
    2. After that, the user may proceed with "Nixellar Service : Type C" described above.
  • Click here for a comprehensive guide.
